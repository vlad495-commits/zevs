## Что такое Stitch?

**Stitch** (stitch.withgoogle.com) — AI-инструмент от Google Labs для генерации UI (интерфейсов) мобильных приложений и веб-страниц. Он создаёт готовый, визуально полноценный дизайн по текстовому описанию.

**Это НЕ**:
- Полноценный конструктор сайтов (не заменяет Tilda/Webflow)
- Инструмент для деплоя (не публикует сайты)
- Замена Figma (хотя имеет плагин)

**Это ДА**:
- Генератор UI-прототипов по описанию
- Быстрый способ получить отправную точку для дизайна
- Экспортёр готового HTML/CSS/React кода

---

## Как устроен интерфейс

### Главный экран
- **Поле ввода** — описание того, что хочешь создать
- **Переключатель** — "Приложение" (мобильный) или "Веб-интерфейс"
- **Модель** — по умолчанию Gemini 3.0 Flash, можно менять
- **Примеры** в левой панели — готовые демо для вдохновения

### После генерации
- **Preview** — визуальный просмотр результата
- **Code** — исходный код (HTML/CSS или React)
- **Edit** — чат для доработок текстом
- **Export** — скачать код, открыть в Figma

---

## Правила и ограничения

1. **Только UI** — Stitch делает внешний вид, не логику/базы данных
2. **Не сохраняет данные** — нет бэкенда, форм, которые реально отправляют письма
3. **Beta-статус** — инструмент постоянно меняется, возможны баги
4. **Лимиты генерации** — в бесплатном плане ограниченное количество запросов
5. **Русский язык** — понимает, но лучше работает на английском
6. **Данные** — Google использует чаты для улучшения модели (предупреждение в интерфейсе)

## Фишки и возможности

### 1. Итеративная доработка
После первой генерации можно писать в чат уточнения:
- "Сделай кнопку крупнее"
- "Измени цвет на тёмно-зелёный"
- "Добавь секцию с отзывами"
- "Убери лишние блоки"

### 2. Переключение между устройствами
Можно посмотреть как выглядит на телефоне и на десктопе.

### 3. Экспорт кода
- **HTML/CSS** — статичная страница
- **React** — компоненты для разработки
- Код можно копировать и дорабатывать вручную (в т.ч. в Claude Code)

### 4. Плагин Figma
Интеграция позволяет:
- Открывать сгенерированный дизайн прямо в Figma
- Редактировать элементы как полноценные компоненты
- Экспортировать в дизайн-систему
- Передавать дизайн разработчику с Dev Mode

### 5. Примеры для вдохновения
В левой панели — 7+ готовых примеров: дашборды, маркетплейсы, приложения.

## Как писать промпты для Stitch

### Структура хорошего промпта

```
[ЧТО ЭТО] + [ДЛЯ КОГО] + [СТИЛЬ] + [КЛЮЧЕВЫЕ СЕКЦИИ] + [ЦВЕТА/ШРИФТЫ]
```

### Плохой промпт ❌
```
Сделай сайт для компании по дезинсекции
```

### Хороший промпт ✅
```
Create a premium landing page for a pest control company "Dez2.0" in Saint Petersburg.
Style: dark background, lime green accent (#A3D044), modern and trustworthy.
Sections: hero with headline and CTA button, trust stats (1200+ clients, 5 years, guarantee), 
services grid (cockroaches, bedbugs, rats, disinfection), how it works (4 steps), 
reviews (3 cards), pricing (3 plans), contact form.
Typography: Manrope font. Mobile responsive.
```

### Ключевые слова для дезинсекции
- `pest control`, `disinsection`, `exterminator`
- `professional`, `certified`, `guaranteed`
- `dark/light theme`, `minimal`, `premium`, `eco-friendly`
- `trust badges`, `social proof`, `before/after`

## Лучшие практики

### Итерация по частям
Не пытайся сгенерировать всё сразу. Лучше:
1. Сначала — общий лейаут и герой
2. Затем — добавь секции по одной
3. Доработай цвета и типографику
4. Экспортируй и дорабатывай в коде

### Указывай референсы
"В стиле Apple.com — минималистично, много воздуха"
"Как у крутых SaaS-сайтов — тёмный фон, яркий акцент"
"Тёплые оттенки, органический стиль как у экологических компаний"

### Уточняй детали CTA
- "Кнопка 'Вызвать мастера' — главный призыв к действию"
- "Телефонный номер — в шапке и в нижнем блоке"
- "Форма: только имя и телефон, не перегружай"

### Работа с Stitch + Claude Code
Оптимальный воркфлоу:
1. **Stitch** → генерирует визуальный прототип
2. **Экспорт HTML** → получаем базовый код
3. **Claude Code** → дорабатывает, добавляет анимации, оптимизирует
4. **Figma плагин** → финальный дизайн для согласования

## Типичные ошибки при работе со Stitch

1. **Слишком общий промпт** — получишь шаблонный результат
2. **Пытаться сделать всё за раз** — лучше итерировать
3. **Игнорировать мобильную версию** — проверяй переключателем
4. **Не экспортировать код** — код нужно брать и дорабатывать
5. **Ожидать готового сайта** — это прототип, не финальный продукт

## Применение для проекта Дез2.0

- Героя с заголовком и кнопкой вызова

- Оптимизация скорости загрузки
- SEO-мета теги
- Реальная отправка форм
- Адаптация под мобильные
